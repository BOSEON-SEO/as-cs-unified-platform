# A/S & CS 통합 관리 플랫폼

## 개요
기존 어드민 시스템에서 A/S 도메인을 분리한 전용 통합 앱.
Electron(Windows/macOS 데스크톱)과 웹(모바일 브라우저)을 동시에 지원하는 React 기반 앱.

## 아키텍처
- **프론트엔드**: React + Tailwind CSS (반응형 레이아웃)
- **데스크톱 래핑**: Electron
- **백엔드**: 기존 Spring Boot API 재사용·확장 (클라이언트→DB 직접 쿼리 금지)
- **DB**: 기존 MySQL 통합 DB (공유, 스키마 추가 방향만 허용)
- **AI 의사결정 서포트**: 로컬 Claude API (Tailscale VPN 경유) — suggestion 전용, 자동 발송 X
- **알림 채널**: 기존 어드민 SMTP, SMS(알림톡), 택배사 API 재사용
- **외부 연동**: 젠데스크는 과거 이력 조회용으로만 (Bearer Token), 신규 티켓은 본 시스템에서 직접 등록

## 사용자 및 권한
- A/S 담당자 (5명): A/S 접수·진행·종료·출고
- CS/CX (4명): 상담 내역·티켓 조회
- 팀장 (2명): 티켓 배분·비용 현황·리포트
- 총괄 (1명): 전체 대시보드·설정

## 주요 도메인 엔티티
- **Customer**: 전화번호(조회 PK 기준)·신상·구매 이력
- **Ticket**: A/S 접수부터 종료까지의 이력 단위 (단계: 접수 → 배정 → 진단/처리결정 → 비용 → 출고/회수 → 종료)
- **Product**: 모델명·시리얼·정품등록 정보
- **Shipment**: 출고·회수 물류 정보
- **CostEntry**: 비용 3축 (우리 지출 / 고객 수납 / 본사 보상)
- **MailLog**: 이메일·SMS 송수신 기록
- **AssignmentRule**: 자동 배정 풀(크루별 ON/OFF) + 모드(라운드 로빈/최소 부하/제품군 매칭) + 제외 조건
- **AILog**: AI 호출 이력 (유형·대상·호출자·채택 여부) — 감사 가능

## 도메인 규칙
- 모든 고객 조회는 **전화번호** 기준
- 티켓은 담당자 단위로 배분되며 상태 이력이 남아야 함
- **자동 배정**: 토글 ON 시 인입 즉시 자동 분배 (배정 풀에 포함된 크루만 대상). 토글 OFF면 미배정 큐에 적재
- **티켓 목록 뷰**: 담당자는 **내 담당 / 우리 팀 / 전체** 3단계 토글로 조회 가능
- **A/S 상세는 단계별 진행** — 6단계 스테퍼로 시각화, 각 단계는 명시적 완료 액션 필요
- 비용은 3축(지출·수납·보상)으로 분리 추적
- **AI 출력은 suggestion 형태만** (자동 발송 절대 금지). 가격·보상·클레임 대응 자동화 금지. PII 마스킹 후 호출
- 회수 관리는 판매 후 단기 접수 건(직접 기사 배정)만 해당
- 교환·반품 처리 시 재고 재등록 / 리퍼브 / 부품 처리 분기 필수
- 이메일·SMS 발송 시 반드시 MailLog 기록
- **리포트 모듈은 설계 중** — 정식 도입 전까지는 전체 통계 페이지에서 임시 다운로드만 제공

## 핵심 페이지 목록 (17개 · v2 prototype 기준)
1. 고객 정보 조회
2. 전체 상담 내역
3. 젠데스크 조회 *(외부 API · 과거 이력 조회용. 마이그레이션 X)*
4. **A/S 티켓 목록** *(내 담당 / 우리 팀 / 전체 토글)*
5. A/S 상세 *(단계별 스테퍼 6단계)*
6. A/S 현황 대시보드
7. A/S 비용 추적
8. A/S 출고 관리
9. 회수 관리
10. 정품등록·쿠폰·시리얼 관리
11. 교환·반품 관리
12. 이메일·SMS 로그
13. 티켓 배분 *(자동 배정 토글 + 크루 풀 ON/OFF)*
14. **AI 의사결정 서포트** *(증상 분석·답변 초안·유사 케이스·고객 컨텍스트)*
15. 리포트 *(설계 중 — Phase 6에서 정식화)*
16. 전체 통계 대시보드
17. 설정

## DoD 패턴
- 각 페이지: 목록 → 상세 → 생성/편집 플로우 완성
- API 호출 전부 Spring 백엔드 경유
- 이메일·SMS 발송 시 MailLog 자동 생성 확인
- AI 호출 시 AILog 자동 기록 + 채택 여부 추적
- 자동 배정 발생 시 ENGINEER_ASSIGNED 이벤트 자동 기록
- Electron 빌드 및 모바일 웹 반응형 양쪽 QA 완료
