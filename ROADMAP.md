## Phase 1: 목업 슬라이드 제작 및 실무진 컨펌

**목표**: 전체 UI 흐름을 Google Slides로 시각화하여 실무진 컨펌 획득

**주요 산출물**
- 전체 15개 페이지 와이어프레임 슬라이드
- 각 페이지 사용 시나리오 및 동작 흐름 설명
- 현재 어드민 시스템 대비 변경·추가 포인트 비교표

**DoD**
- A/S·CS/CX·팀장·총괄 전 사용자 컨펌 완료
- 피드백 반영 후 최종 슬라이드 확정

## Phase 2: 기반 구축 및 고객 조회

**목표**: 프로젝트 부트스트랩, 인증, 기본 고객 조회 기능 동작

**주요 산출물**
- React + Electron 프로젝트 설정 (개발·빌드 환경)
- Spring API 연동 레이어 및 HTTP 클라이언트 구성
- 로그인·권한 시스템 (역할별 접근 제어)
- 고객 전화번호 조회 페이지 (구매 이력·상담 이력 표시)
- 전체 대시보드 뼈대

**DoD**
- Electron 창에서 로그인 후 고객 조회 가능
- 모바일 브라우저에서도 동일 기능 접근 가능

## Phase 3: A/S 핵심 플로우

**목표**: A/S 접수-진행-종료 사이클 및 티켓 배분 완성

**주요 산출물**
- A/S 접수 폼 (고객+제품 매칭, 증상·처리 메모)
- A/S 상세 페이지 (진행 상태, 처리 이력, 비용 산정)
- 티켓 목록·배분 (담당자 배정 및 상태 변경)
- A/S 비용 추적 (지출·수납·본사보상 3축)
- 리포트 현황 (본사 보상 금액 추적)

**DoD**
- 접수 → 처리 → 완료 E2E 플로우 정상 동작
- 비용 3축 합산 수치 정확

## Phase 4: 물류·회수·반품 및 정품등록

**목표**: 출고·회수·교환반품 물류 플로우 및 정품등록 완성

**주요 산출물**
- A/S 출고 관리 (수신인 입력 → CX 인계)
- 회수 추적 (기사 배정, 미회수 건 대시보드)
- 교환&반품 처리 (재고 재등록·리퍼브·부품 분기)
- 택배사 API 연동
- 정품등록·시리얼 검수·쿠폰 발급 관리

**DoD**
- 출고 생성 후 CX 인계 상태 반영
- 미회수 건 필터링 가능
- 교환·반품 완료 후 재고 상태 정상 갱신

## Phase 5: 커뮤니케이션 통합 및 LLM 위키 연동

**목표**: 이메일·SMS 로그, 젠데스크 대체, Claude LLM 자동 답변 제안 완성

**주요 산출물**
- 이메일·SMS 송수신 UI 및 MailLog 자동 기록
- 젠데스크 문의 마이그레이션 UI
- Claude LLM 위키 연동 (Tailscale 경유, 답변 제안 패널)
- 전체 페이지 반응형 QA 및 Electron 빌드 검증

**DoD**
- 이메일 발송 시 MailLog 자동 생성
- LLM 답변 제안 표시 (자동 발송 없음)
- Electron 빌드 및 모바일 웹 양쪽 최종 QA 통과
