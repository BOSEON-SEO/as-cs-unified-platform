# 06. A/S 비용 추적

**비용 3축 분리 추적 — 지출 / 수납 / 보상**

Phase 1 산출물 | 슬라이드 06 / 15

---

## 목적 & 핵심 가치

### 왜 만드나?

- A/S 비용을 **3축(우리 지출 / 고객 수납 / 본사 보상)**으로 분리하여 정확한 손익 파악
- 티켓별 비용 입력부터 월별 집계까지 **하나의 페이지에서 관리**
- 기존 엑셀 수동 관리 → **시스템 기반 자동 집계 및 승인 워크플로**

### 기존 vs 신규

| 기존 어드민 | 신규 통합 앱 |
|------------|-------------|
| 엑셀 수동 집계 | 시스템 내 자동 집계 |
| 비용 축 구분 없음 | 3축(지출/수납/보상) 분리 |
| 승인 프로세스 없음 | 팀장 승인 워크플로 |
| 월말 수동 정산 | 실시간 월별 집계 테이블 |

---

## 접근 권한

| 역할 | 인원 | 접근 범위 | 제한 사항 |
|------|------|----------|----------|
| **A/S 담당자** | 5명 | 본인 담당 티켓 비용 입력/수정 | 타 담당자 건 수정 불가, 삭제 불가 |
| **CS/CX** | 4명 | 조회 불가 | 접근 권한 없음 |
| **팀장** | 2명 | 전체 조회 + 승인 | 직접 입력 불가 |
| **총괄** | 1명 | 전체 조회 + 수정/삭제 | 제한 없음 |

## 비용 3축 상세

| 축 | 분류 | 항목 | 설명 |
|----|------|------|------|
| **우리 지출** | 부품비 | 교체 부품 원가 | 자사 부담 비용 |
| | 택배비 | 발송/회수 택배비 | 왕복 발생 시 각각 기록 |
| | 수리비 | 외주 수리 비용 | 외부 수리 위탁 시 |
| **고객 수납** | 수리비 청구 | 유상 수리 청구액 | 고객에게 청구하는 금액 |
| | 택배비 청구 | 착불 택배비 | 고객 부담 택배비 |
| **본사 보상** | 클레임 보상금 | 본사 보상 수령액 | 제조사 클레임 처리 보상 |
| | 무상 수리 보전 | 보증기간 수리 보전 | 본사 무상 수리 비용 보전 |

## 비용 입력 폼 상세

| UI 요소 | 타입 | 상세 |
|---------|------|------|
| 비용 축 선택 | radio | 지출 / 수납 / 보상 (필수) |
| 비용 항목 | select | 축 선택에 따라 항목 동적 변경 |
| 금액 | number | 필수, 원 단위, 천 단위 콤마 표시 |
| 비고 | textarea | 선택, 비용 발생 사유 |
| 증빙 파일 | file | 선택, 영수증/견적서 첨부 |
| 저장 | btn-primary | POST /api/tickets/{id}/costs |

### 승인 워크플로

```
A/S 담당자 비용 입력 → 상태: "대기"
    ↓
팀장 검토 → [승인] / [반려(사유 입력)]
    ↓
승인 시: 상태 "승인" → 월별 집계에 반영
반려 시: 상태 "반려" → 담당자 재입력
```

## 시나리오 SC-1: A/S 담당자 비용 입력

```
A/S 담당자 → 티켓 상세(#4)에서 [비용 추가] 클릭
    ↓
비용 입력 모달 표시
    ↓
축: "우리 지출" → 항목: "부품비" → 금액: 85,000원
    ↓
POST /api/tickets/{id}/costs
  body: { axis: "expense", category: "parts", amount: 85000 }
    ↓
토스트: "비용이 등록되었습니다 (승인 대기)"
    ↓
비용 추적 페이지 목록에 "대기" 상태로 표시
```

---

## 시나리오 SC-2: 팀장 월별 비용 점검 및 승인

```
팀장 로그인 → A/S 비용 추적 진입
    ↓
기간 필터: "2026년 4월" → [적용]
    ↓
API: GET /api/costs/summary?month=2026-04
    ↓
월별 집계 확인 → 순비용 250천원
    ↓
상태 "대기" 건 필터 → 미승인 목록 확인
    ↓
각 건 [승인] 또는 [반려] 처리
    → PATCH /api/tickets/{id}/costs/{costId}
    → 승인 시 집계 자동 갱신
```

## API 엔드포인트 매핑

| 액션 | Method | Endpoint |
|------|--------|----------|
| 티켓별 비용 목록 | GET | `/api/tickets/{id}/costs` |
| 비용 입력 | POST | `/api/tickets/{id}/costs` |
| 비용 수정 | PATCH | `/api/tickets/{id}/costs/{costId}` |
| 비용 삭제 | DELETE | `/api/tickets/{id}/costs/{costId}` (총괄만) |
| 월별 집계 | GET | `/api/costs/summary?month={YYYY-MM}` |
| 비용 승인/반려 | PATCH | `/api/tickets/{id}/costs/{costId}` |
| 비용 목록 (전체) | GET | `/api/costs?from&to&axis&assignee&status` |
