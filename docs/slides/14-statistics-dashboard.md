# 14. 전체 통계 대시보드

**전사 KPI + 운영 진단 + 액션 필요 큐**

Phase 1 산출물 | 슬라이드 14 / 15

---

## 목적 & 핵심 가치

### 왜 만드나?

- 전사 A/S 운영 상태를 **KPI 5섹션 + 트렌드 차트**로 실시간 모니터링
- tobe-tickets `/analytics` 운영 진단 기능을 흡수하여 **데이터 품질 관리** 통합
- tobe-tickets `/actions` 11개 패턴(좀비·응대누락·가짜완료)을 **액션 필요 큐**로 제공
- **30초 자동 새로고침**으로 실시간 운영 현황 파악

### 기존 vs 신규

| 기존 어드민 | 신규 통합 앱 |
|------------|-------------|
| 통계 기능 없음 — DB 직접 쿼리 필요 | 실시간 대시보드 내장 |
| tobe-tickets 별도 앱에서 운영 진단 | 통합 앱에 KPI + 진단 흡수 |
| 이상 패턴 수동 확인 | 11개 액션 패턴 자동 감지·큐 |
| 새로고침 수동 | 30초 자동 갱신 |

## UI 상세

### KPI 카드 영역 [A]
- **4개 핵심 지표**: 총 티켓 수, 완료율(%), 평균 처리일, 비용 합계(₩)
- 각 카드에 전월 대비 증감률 표시 (▲ 초록 / ▼ 빨강)
- 우측 상단 **Live 인디케이터** + 30초 자동 새로고침 카운트다운
- 수동 새로고침 버튼 [🔄]

### 월별 트렌드 [B]
- 라인 차트: 접수 vs 완료 건수 (최근 6개월)
- 호버 시 상세 수치 툴팁

### 제품별 A/S 빈도 [C] + 증상 유형 분포 [D]
- 수평 바 차트: 상위 10개 모델 A/S 건수
- 도넛 차트: 증상 유형별 비율 (전원·소음·파손·외관·기타)

### 액션 필요 큐 [E] — tobe-tickets `/actions` 흡수
- 11개 패턴 분류: 좀비 티켓, 응대 누락, 가짜 완료, 장기 미배정, SLA 초과 등
- 심각도별 색상 코딩 (🔴 Critical / 🟠 High / 🟡 Medium)
- 클릭 시 해당 티켓 목록으로 이동

## 시나리오

### 시나리오 1: 총괄의 아침 운영 점검
1. 총괄이 대시보드 진입 → KPI 4개 카드 자동 로딩 (30초 갱신)
2. 완료율 ▼ 하락 확인 → 월별 트렌드 차트에서 이번 달 접수 급증 파악
3. 액션 필요 큐에서 **좀비 티켓 7건** 확인 → 클릭하여 해당 티켓 목록 이동
4. 담당자별 배분 불균형 확인 → 티켓 배분 페이지로 이동

### 시나리오 2: 팀장의 제품별 A/S 패턴 분석
1. 팀장이 제품별 A/S 빈도 차트 확인
2. 모델A가 비정상적으로 높은 A/S 건수 → 증상 유형 분포에서 `전원 불량 35%` 확인
3. 해당 모델 리포트 생성을 위해 리포트 페이지로 이동

### 시나리오 3: 액션 필요 큐 처리
1. 팀장이 **응대 누락 3건** 클릭
2. 48시간 이상 응대 없는 티켓 목록 표시
3. 각 티켓에 담당자 재배정 또는 우선순위 상향 처리

1. tobe-tickets `/analytics` 5개 KPI 섹션 — 전부 흡수 vs 일부만?
