# 05. A/S 현황 대시보드

**실시간 A/S 처리 현황 및 KPI 모니터링**

Phase 1 산출물 | 슬라이드 05 / 15

---

## 목적 & 핵심 가치

### 왜 만드나?

- A/S 처리 현황을 **실시간 KPI 카드 + 차트**로 한눈에 파악
- 기존 tobe-tickets 개요 페이지의 6개 KPI·처리유형 분포·단계 평균을 **흡수·통합**
- 팀장/총괄이 **담당자별 업무 부하와 병목**을 즉시 확인

### 기존 vs 신규

| 기존 어드민 | 신규 통합 앱 |
|------------|-------------|
| tobe-tickets 개요 페이지 (6 KPI) | 통합 대시보드로 흡수 |
| 엑셀 수동 집계 | 실시간 API 기반 자동 집계 |
| 담당자별 현황 없음 | 담당자별 바차트 제공 |
| 주간 트렌드 없음 | 주간 트렌드 라인차트 |

---

## 접근 권한

| 역할 | 인원 | 접근 범위 | 제한 사항 |
|------|------|----------|----------|
| **A/S 담당자** | 5명 | 본인 담당 건만 | 타 담당자 건 조회 불가 |
| **CS/CX** | 4명 | 전체 조회 | 조회만 가능, 필터 제한 |
| **팀장** | 2명 | 전체 조회 + 담당자 필터 | 데이터 수정 불가 |
| **총괄** | 1명 | 전체 조회 + 모든 필터 | 제한 없음 |

## KPI 카드 영역 [B] 상세

| UI 요소 | 타입 | 상세 |
|---------|------|------|
| 금일 접수 | KPI 카드 | 오늘 생성된 티켓 수, 전일 대비 증감 표시 |
| 진행중 | KPI 카드 | 상태가 접수/진행 중인 전체 티켓 수 |
| 완료 | KPI 카드 | 오늘 완료 처리된 티켓 수 |
| 평균 처리일 | KPI 카드 | 접수~완료 평균 소요 일수, 목표 대비 색상 |

## 시나리오 SC-1: 팀장 주간 현황 점검

```
팀장 로그인 → A/S 현황 대시보드 진입
    ↓
기간 필터: "이번 주" 선택 → [적용]
    ↓
API: GET /api/dashboard/as-summary?from=2026-04-22&to=2026-04-28
    ↓
KPI 카드 4종 + 차트 3종 갱신
    ↓
담당자별 바차트에서 특정 담당자 클릭
    ↓
해당 담당자 건만 필터링 → 상세 티켓 목록 표시
    ↓
병목 건 확인 → 티켓 상세(#4)로 이동
```

---

## 시나리오 SC-2: 총괄 월간 트렌드 분석

```
총괄 로그인 → A/S 현황 대시보드 진입
    ↓
기간 필터: "이번 달" 선택 → [적용]
    ↓
API: GET /api/dashboard/as-trend?from=2026-04-01&to=2026-04-28
    ↓
주간 트렌드 라인차트 → 접수/완료 추이 확인
    ↓
처리유형 분포(파이) → 수리/교환/반품 비율 확인
    ↓
단계별 평균 소요 → 병목 단계 식별
    ↓
리포트 현황(#13)으로 이동하여 상세 분석
```

# 확인 필요 사항

- **KPI 목표치 설정**: 평균 처리일 목표 (예: 3일 이내) → 팀장/총괄 확정 필요
- **tobe-tickets 개요 완전 대체 여부**: 기존 페이지 폐기 vs 병행 → 실무진 컨펌
- **차트 자동 갱신 주기**: 실시간 vs 5분 간격 → 성능 고려 필요
