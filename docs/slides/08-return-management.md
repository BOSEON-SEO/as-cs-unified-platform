# 08. 회수 관리

**판매 후 단기 접수 건 — 직접 기사 배정 회수**

Phase 1 산출물 | 슬라이드 08 / 15

---

## 목적 & 핵심 가치

### 왜 만드나?

- 판매 후 단기 접수 건(초기 불량 등)은 **택배 회수가 아닌 직접 기사 배정**으로 빠르게 수거
- 회수 요청 → 기사 배정 → 수거 완료까지 **상태 추적 체계화**
- 미회수 건 대시보드로 **지연 건 즉시 파악 및 대응**

### 기존 vs 신규

| 기존 어드민 | 신규 통합 앱 |
|------------|-------------|
| 기사 배정 구두/메신저 전달 | 시스템 내 기사 배정 관리 |
| 회수 상태 수동 확인 | 요청→배정→수거완료 상태 추적 |
| 미회수 건 파악 어려움 | 미회수 건 대시보드 제공 |
| 기사별 현황 없음 | 기사별 배정 현황 조회 |

## 접근 권한

| 역할 | 인원 | 접근 범위 | 제한 사항 |
|------|------|----------|----------|
| **A/S 담당자** | 5명 | 회수 요청 생성 + 기사 배정 | 삭제 불가 |
| **CS/CX** | 4명 | 조회만 | 생성/수정 불가 |
| **팀장** | 2명 | 전체 조회 + 미회수 모니터링 | 기사 배정 가능 |
| **총괄** | 1명 | 전체 조회 + 수정/삭제 | 제한 없음 |

## 회수 요청 폼 상세

| UI 요소 | 타입 | 상세 |
|---------|------|------|
| 연결 티켓 | auto | 티켓 번호 자동 연결 |
| 고객명 | text | 자동 채움 (읽기 전용) |
| 연락처 | text | 자동 채움 (읽기 전용) |
| 수거 주소 | text | 필수, 고객 주소 자동 채움 + 직접 입력 |

## 기사 관리 및 미회수 대시보드 상세

### 기사별 현황

| UI 요소 | 상세 |
|---------|------|
| 기사 목록 | 이름, 연락처, 담당 지역 |
| 배정 건수 | 기사별 현재 배정 중인 건수 |
| 완료 건수 | 기사별 금일/주간 완료 건수 |
| 부하 표시 | 배정 건수에 따른 색상 (3건 이하: 녹색, 4~6건: 노란색, 7건+: 빨간색) |

### 미회수 대시보드 [B]

| KPI | 설명 | 알림 조건 |
|-----|------|----------|
| 미회수 건 | 상태가 요청/배정인 전체 건수 | 7건 이상 시 빨간색 |
| 금일 배정 | 오늘 기사 배정된 건수 | - |
| 금일 완료 | 오늘 수거 완료된 건수 | - |
| 지연 건 | 희망 수거일 초과 미회수 건 | 1건 이상 시 팀장 알림 |

## 시나리오 SC-1: A/S 담당자 회수 요청 및 기사 배정

```
A/S 담당자 → 티켓 상세(#4)에서 [회수 요청] 클릭
    ↓
회수 요청 폼 표시 (고객/제품 정보 자동 채움)
    ↓
기사: "박기사" 선택, 희망 수거일: "2026-04-29"
    ↓
POST /api/returns
  body: { ticketId, customer, address, product, driverId, pickupDate }
    ↓
토스트: "회수 요청이 등록되었습니다"
    ↓
기사에게 배정 알림 발송 (SMS)
    → POST /api/mail-logs/sms → MailLog 기록
    ↓
회수 목록에 "배정" 상태로 표시
```

## 시나리오 SC-2: 팀장 미회수 건 모니터링

```
팀장 로그인 → 회수 관리 진입
    ↓
미회수 대시보드 KPI 확인: 미회수 7건, 지연 2건
    ↓
API: GET /api/returns/pending
    ↓
상태 필터: "요청" (미배정 건) 확인
    ↓
미배정 건 선택 → [기사 배정] 클릭
    ↓
PATCH /api/returns/{id}
  body: { driverId: "D003", pickupDate: "2026-04-29" }
    ↓
지연 건 확인 → 담당자에게 독촉 / 기사 재배정
    ↓
토스트: "기사 배정 완료"
```

## API 엔드포인트 매핑

| 액션 | Method | Endpoint |
|------|--------|----------|
| 회수 목록 | GET | `/api/returns?from&to&status&driverId` |
| 회수 요청 생성 | POST | `/api/returns` |
| 회수 수정 (기사 배정 포함) | PATCH | `/api/returns/{id}` |
| 회수 삭제 | DELETE | `/api/returns/{id}` (총괄만) |
| 미회수 건 조회 | GET | `/api/returns/pending` |
| 기사 목록 | GET | `/api/drivers` |
| 기사별 현황 | GET | `/api/drivers/{id}/returns` |

# 확인 필요 사항

- **단기 접수 기준**: 판매 후 며칠 이내를 "단기"로 정의할지 → 실무진 확정 필요
- **기사 관리 범위**: 기사 등록/수정은 이 페이지 vs 설정(#15) → 확정 필요
