# 티켓 배분·담당자 관리

## Phase 1 산출물 | 슬라이드 12 / 15

A/S & CS 통합 관리 플랫폼

# 목적 & 핵심 가치

### 왜 만드나?

미배정 티켓을 실시간으로 파악하고, 담당자별 업무량을 기반으로 **수동 배정(드래그&드롭)** 또는 **자동 배정(라운드로빈/역량 기반)**을 수행한다. 기존 tobe-tickets/team 담당자 현황 기능을 흡수·통합한다.

| 구분 | 기존 시스템 | 신규 시스템 |
|------|------------|------------|
| 티켓 배정 | 구두 전달 또는 메신저 기반 수기 배정 | 시스템 내 수동/자동 배정 + 이력 추적 |
| 업무량 파악 | 별도 시트에서 수기 집계 | 담당자별 실시간 보유 건수 대시보드 |
| 자동 배정 | 미지원 | 라운드로빈 / 역량 기반 자동 배정 규칙 |
| 담당자 현황 | tobe-tickets/team 별도 페이지 | 통합 화면에서 한눈에 확인 |

# 접근 권한

| 역할 | 인원 | 접근 범위 | 제한 사항 |
|------|------|----------|----------|
| A/S 담당자 | 5명 | 본인 배정 건 조회 | 타 담당자 재배정·자동 배정 설정 불가 |
| CS/CX | 4명 | 본인 배정 건 조회 | 배정·재배정 불가 |
| 팀장 | 2명 | 미배정 큐 관리, 수동/자동 배정, 재배정, 업무량 조회 | 없음 |
| 총괄 | 1명 | 전체 기능 + 자동 배정 정책 설정 | 없음 |

# UI 요소 상세 — 미배정 큐

| 요소 | 유형 | 설명 |
|------|------|------|
| 미배정 티켓 카드 | 카드 (드래그 가능) | 티켓번호, 고객명, 제품, 유형, 접수일시 표시 |
| 드래그&드롭 배정 | 드래그 인터랙션 | 카드를 담당자 영역으로 드래그하여 배정 (데스크톱) |

| 일괄 선택 배정 | 체크박스 + 버튼 | 여러 티켓 선택 후 한 명에게 일괄 배정 |
| 접수 시간 표시 | 텍스트 + 경고 뱃지 | 미배정 2시간 초과 시 주황, 4시간 초과 시 빨강 뱃지 |
| 정렬 | 드롭다운 | 접수일시순 / 긴급도순 / 제품유형순 |

# UI 요소 상세 — 담당자 현황·자동 배정

| 요소 | 유형 | 설명 |
|------|------|------|
| 담당자별 보유 건수 | 프로그레스 바 | 현재 보유 건수 / 최대 보유 건수 비율 표시 |

# 시나리오 1: 수동 배정 (드래그&드롭)

```
팀장                  시스템
  │                    │
  ├─ 티켓 배분 화면 접근 →│
  │                    ├─ 미배정 큐 로딩 (12건)
  │                    ├─ 담당자별 보유 건수 표시
  │                    │
  │←── 화면 표시 ────────┤
  │                    │
  ├─ #TK-0401 카드를     │
  │  "정담당(2건)"에      │
  │  드래그&드롭 ────────→│
  │                    ├─ 배정 처리
  │                    │  - Ticket.assignee = 정담당
  │                    │  - 상태 이력 기록
  │                    │  - 미배정 큐: 12→11건
  │                    │  - 정담당: 2→3건
  │                    │
  │←── 배정 완료 ─────────┤
  │  (정담당에게 알림 발송) │
  │                    ├─ MailLog 기록
```

# API 엔드포인트 매핑

| 메서드 | 엔드포인트 | 설명 | 권한 |
|--------|-----------|------|------|
| GET | `/api/tickets/unassigned` | 미배정 티켓 목록 조회 | 팀장 이상 |
| POST | `/api/tickets/{id}/assign` | 티켓 수동 배정 (담당자 지정) | 팀장 이상 |
| POST | `/api/tickets/batch-assign` | 다건 일괄 배정 | 팀장 이상 |
| POST | `/api/tickets/{id}/reassign` | 티켓 재배정 (사유 필수) | 팀장 이상 |
| GET | `/api/assignees/workload` | 담당자별 보유 건수 현황 | 팀장 이상 |
| GET | `/api/assignment-rules` | 자동 배정 규칙 조회 | 팀장 이상 |
| PUT | `/api/assignment-rules` | 자동 배정 규칙 수정 | 총괄 |
| POST | `/api/assignment-rules/execute` | 자동 배정 수동 실행 | 팀장 이상 |
| GET | `/api/tickets/{id}/assignment-history` | 배정 이력 조회 | 전체 |

# 확인 필요 사항

1. 자동 배정 대상: A/S 담당자(5명)만인지, CS/CX(4명)도 포함하는지
2. 역량 기반 배정 시 제품 카테고리-담당자 매핑 초기 데이터
3. 재배정 시 사유 입력 필수 여부 (필수 vs 선택)
4. 미배정 경고 임계값 (2시간/4시간 기준 조정 가능 여부)
5. tobe-tickets/team 기존 데이터 마이그레이션 범위
