# 목적 & 핵심 가치

### 왜 만드나?

정품등록·시리얼·쿠폰을 하나의 화면에서 통합 관리하여 **위변조 방지**, **보증 기간 자동 산정**, **쿠폰 자동 발급**까지 일관된 워크플로우를 제공한다.

| 구분 | 기존 시스템 | 신규 시스템 |
|------|------------|------------|
| 정품등록 조회 | 어드민 내 분산된 메뉴에서 개별 확인 | 통합 목록 + 상세 화면에서 일괄 관리 |
| 시리얼 검증 | 수동 대조 또는 별도 시트 확인 | 실시간 중복/위변조 자동 체크 API |
| 쿠폰 발급 | 수동 발급 후 별도 기록 | 정품등록 완료 시 쿠폰 자동 발급 |
| 보증 만료 관리 | 구매일 기준 수기 계산 | 구매일 기반 보증만료일 자동 산정·표시 |

# 접근 권한

| 역할 | 인원 | 접근 범위 | 제한 사항 |
|------|------|----------|----------|
| A/S 담당자 | 5명 | 정품등록 목록·상세 조회, 시리얼 검증, 쿠폰 발급 이력 조회 | 쿠폰 수동 무효화 불가 |
| CS/CX | 4명 | 정품등록 목록·상세 조회 (읽기 전용) | 등록·수정·쿠폰 발급 불가 |
| 팀장 | 2명 | 전체 조회 + 쿠폰 무효화 + 시리얼 블랙리스트 관리 | 없음 |
| 총괄 | 1명 | 전체 기능 + 정품등록 정책 설정 | 없음 |

# API 엔드포인트 매핑

| 메서드 | 엔드포인트 | 설명 | 권한 |
|--------|-----------|------|------|
| GET | `/api/registrations` | 정품등록 목록 조회 (필터·페이징) | 전체 |
| GET | `/api/registrations/{id}` | 정품등록 상세 조회 | 전체 |
| POST | `/api/registrations` | 정품등록 신규 생성 | A/S 담당자 이상 |
| PATCH | `/api/registrations/{id}` | 정품등록 상태 변경 (승인/반려) | A/S 담당자 이상 |
| GET | `/api/serials/verify` | 시리얼 검증 (중복·위변조 체크) | 전체 |
| POST | `/api/serials/blacklist` | 시리얼 블랙리스트 등록 | 팀장 이상 |
| POST | `/api/coupons` | 쿠폰 수동 발급 | A/S 담당자 이상 |
| GET | `/api/coupons` | 쿠폰 목록 조회 | 전체 |
| PATCH | `/api/coupons/{id}/invalidate` | 쿠폰 무효화 | 팀장 이상 |

# 확인 필요 사항

1. 정품등록 시 구매 증빙 첨부 필수 여부
2. 쿠폰 유효 기간 정책 (등록일 기준 6개월? 12개월?)
3. 시리얼 블랙리스트 등록 시 기존 쿠폰 자동 무효화 여부
4. 제품별 보증 기간(월) 데이터 관리 주체 (Product 엔티티에 포함?)
5. 정품등록 채널 구분 필요 여부 (웹/앱/오프라인)
