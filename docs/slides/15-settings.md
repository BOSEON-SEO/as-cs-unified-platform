## 목적 & 핵심 가치

### 왜 만드나?

- **사용자 계정** 추가·수정·비활성화 및 역할 배정을 총괄이 직접 관리
- 알림 채널(이메일/SMS/알림톡), 자동배정 규칙, VIP 기준 등 **시스템 설정 일원화**
- LLM 위키(Claude API, Tailscale VPN) 연결 설정 및 **답변 제안 on/off** 제어
- 비용 항목·상태 코드·택배사 목록 등 **기본값(마스터 데이터)** 관리

### 기존 vs 신규

| 기존 어드민 | 신규 통합 앱 |
|------------|-------------|
| 사용자 관리 — 개발팀 DB 직접 수정 | 총괄 UI에서 직접 관리 |
| 알림 설정 — 코드 하드코딩 | 설정 화면에서 동적 변경 |
| LLM 연동 없음 | Claude API 연결 + 제안 on/off |
| 기본값 변경 시 배포 필요 | 설정 화면에서 즉시 반영 |

## UI 상세

### 설정 탭 [A]
- **4개 탭**: 사용자 | 시스템 | LLM 위키 | 기본값
- 탭 전환 시 해당 영역만 렌더링 (lazy load)

### 사용자 계정 관리 [B]
- 테이블: 이름, 이메일, 역할(A/S 담당자 / CS·CX / 팀장 / 총괄), 상태(활성/비활성)
- **추가**: 모달 — 이름·이메일·역할 입력, 초기 비밀번호 자동 생성
- **편집**: 인라인 또는 모달 — 역할 변경, 비밀번호 초기화
- **비활성화**: 삭제 대신 비활성 처리 (데이터 보존)

### 시스템 설정 [C]
- **알림 채널**: 이메일 SMTP / SMS / 알림톡 — 개별 on/off 토글
- **자동배정 규칙**: 라운드로빈 | 부하 기반 | 수동 — 드롭다운
- **VIP 기준**: 구매 금액 임계값 + 구매 횟수 임계값 (OR 조건)

### LLM 위키 설정 [D]
- Claude API 엔드포인트 (Tailscale VPN 경유 URL)
- API 키 (마스킹 표시, 수정 시 전체 재입력)
- 답변 제안 기능 on/off 토글 (자동 발송 절대 금지 — suggestion만)

### 기본값 관리 [E]
- **비용 항목**: 항목명 추가·수정·삭제 (우리 지출 / 고객 수납 / 본사 보상 카테고리)
- **상태 코드**: 티켓 상태 코드 목록 관리 (접수·진행·보류·완료·취소)
- **택배사 목록**: 택배사명·API 코드 매핑 관리

## 시나리오

### 시나리오 1: 신규 A/S 담당자 추가
1. 총괄이 설정 → 사용자 탭 → `[+ 사용자 추가]` 클릭
2. 이름 `홍길동`, 이메일 `gd@company.com`, 역할 `A/S 담당자` 입력
3. `POST /api/users` → 초기 비밀번호 자동 생성 및 이메일 발송
4. 사용자 목록에 신규 행 추가, 상태 `활성`

### 시나리오 2: 자동배정 규칙 변경
1. 총괄이 시스템 탭 → 자동배정 규칙 `부하 기반` 선택
2. `[저장]` → `PATCH /api/settings/auto-assign` 호출
3. 이후 신규 티켓 자동배정 시 각 담당자의 현재 진행 건수 기반 배분

### 시나리오 3: LLM 답변 제안 비활성화
1. 총괄이 LLM 위키 탭 → 답변 제안 토글 OFF
2. `PATCH /api/settings/llm` → `{ suggestionEnabled: false }`
3. 전체 앱에서 LLM 답변 제안 UI 숨김 처리

### 시나리오 4: 택배사 목록 추가
1. 총괄이 기본값 탭 → 택배사 목록 → `[+ 추가]`
2. 택배사명 `한진택배`, API 코드 `HANJIN` 입력
3. `POST /api/settings/couriers` → 출고 관리에서 선택 가능

## API 매핑

| 동작 | Method | Endpoint | 비고 |
|------|--------|----------|------|
| 사용자 목록 조회 | GET | `/api/users` | 전체 사용자 |
| 사용자 추가 | POST | `/api/users` | 역할·이메일·이름 |
| 사용자 수정 | PATCH | `/api/users/{id}` | 역할 변경, 비활성화 |
| 시스템 설정 조회 | GET | `/api/settings/system` | 알림·자동배정·VIP |
| 시스템 설정 변경 | PATCH | `/api/settings/system` | 부분 업데이트 |
| LLM 설정 조회 | GET | `/api/settings/llm` | API URL, 제안 on/off |
| LLM 설정 변경 | PATCH | `/api/settings/llm` | API 키 마스킹 처리 |
| 기본값 조회 | GET | `/api/settings/defaults?type={type}` | cost-items/statuses/couriers |
| 기본값 추가 | POST | `/api/settings/defaults` | type + payload |
| 기본값 수정 | PATCH | `/api/settings/defaults/{id}` | 개별 항목 수정 |
| 기본값 삭제 | DELETE | `/api/settings/defaults/{id}` | 소프트 삭제 |

## 확인 필요 사항

1. 사용자 인증 방식 — 자체 로그인 vs SSO(구글 워크스페이스) 연동?
2. 비밀번호 정책 — 최소 길이, 복잡도, 만료 주기 기준
3. 자동배정 가중치 — 부하 기반 시 최대 진행 건수 상한선
4. LLM API 키 관리 — 환경 변수 vs DB 저장 (보안 검토 필요)
5. 기본값 삭제 시 — 이미 사용 중인 항목 삭제 방지 로직 필요 여부
6. 알림톡 설정 — 카카오 비즈메시지 발신 프로필 관리 범위
