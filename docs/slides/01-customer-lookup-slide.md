## 1. 슬라이드 제목 및 목적

### 슬라이드 제목
**고객 정보 조회 — 전화번호 기반 통합 고객 뷰**

### 목적
- 전화번호 하나로 고객의 신상, 구매, A/S, 상담, 정품등록 전 이력을 한 화면에 통합 조회
- CS/CX가 전화 인입 시 3초 이내에 고객 컨텍스트를 파악할 수 있는 원스톱 진입점
- 기존 어드민의 분산된 고객 조회(이름 검색, 주문번호 검색, 별도 A/S 조회)를 단일 페이지로 대체

### 접근 가능 역할

| 역할 | 이 페이지 접근 | 제한 사항 |
|------|-------------|----------|
| A/S 담당자 | READ + A/S 접수 CTA | 고객 정보 편집 불가, A/S 탭에서 접수 진입 가능 |
| CS/CX | READ + 상담 기록 추가 | 메모 탭 작성 가능, A/S 접수 CTA 숨김 |
| 팀장 | READ + 담당자 이력 확인 | 전체 탭 조회 가능, 편집 불가 |
| 총괄 | FULL READ | 전체 탭 조회 가능, 고객 정보 편집 가능 |

```

    v

    +-- 결과 1건 --> 고객 카드 [B] + 탭 [C] 즉시 표시

    +-- 결과 2건+ --> 복수 선택 모달 [F] 표시
                        |
                        +-- 고객 카드 클릭 --> 해당 고객 로딩
    |
    v
```

```
  - URL: /as/new?customerId={id}&orderId={orderId}&productId={productId}
  - 고객명, 전화번호, 제품명, 모델명, 시리얼 자동 채움
    |
    v
접수 폼 작성 --> 저장 --> A/S 이력 탭에 새 티켓 반영
```

### SC-3: VIP 고객 응대 (CS/CX)

```
    |

```
