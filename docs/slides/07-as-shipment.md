# 07. A/S 출고 관리

**택배 출고 생성·추적 및 CX 인계 관리**

Phase 1 산출물 | 슬라이드 07 / 15

---

## 목적 & 핵심 가치

### 왜 만드나?

- A/S 처리 후 **출고(교체품·수리품 발송)를 체계적으로 관리**
- 택배사 선택 → 송장번호 입력 → 배달 추적까지 **원스톱 관리**
- CS/CX에 인계가 필요한 건에 **CX 인계 플래그**로 명확한 업무 전달

### 기존 vs 신규

| 기존 어드민 | 신규 통합 앱 |
|------------|-------------|
| 택배사 사이트에서 개별 추적 | 시스템 내 택배 추적 링크 통합 |
| CX 인계 구두/메신저 전달 | CX 인계 플래그 시스템화 |
| 출고 이력 엑셀 관리 | 출고 목록 실시간 관리 |
| 상태 수동 업데이트 | 준비/발송/배달완료 자동 흐름 |

## 출고 생성 폼 [C] 상세

| UI 요소 | 타입 | 상세 |
|---------|------|------|
| 연결 티켓 | auto | 티켓 번호 자동 연결 (티켓 상세에서 진입 시) |
| 수신인명 | text | 필수, 고객명 자동 채움 |
| 연락처 | text | 필수, 고객 전화번호 자동 채움 |
| 배송 주소 | text | 필수, 최근 배송지 자동 채움 + 직접 입력 |
| 택배사 선택 | select | CJ대한통운 / 한진택배 / 로젠택배 / 우체국 / 기타 |
| 송장번호 | text | 택배사 선택 후 입력, 숫자만 허용 |
| CX 인계 | checkbox | 체크 시 CS/CX에 인계 알림 발송 |
| 비고 | textarea | 선택, 배송 관련 메모 |

## 택배 추적 및 CX 인계 상세

### 택배 추적

| UI 요소 | 상세 |
|---------|------|
| 추적 링크 | 택배사별 추적 URL + 송장번호 자동 조합 |
| 추적 팝업 | 클릭 시 택배사 추적 페이지 새 탭 오픈 |
| 상태 자동 갱신 | 택배사 API 연동 (배달완료 시 상태 자동 변경) |

### CX 인계 플래그

| UI 요소 | 상세 |
|---------|------|
| 인계 플래그 | 출고 생성 시 체크박스로 설정 |
| 인계 알림 | CS/CX에 시스템 알림 발송 |
| 인계 확인 | CS/CX가 [확인] 버튼 클릭 → 인계 완료 처리 |
| 필터 | 출고 목록에서 "CX 인계" 필터로 미확인 건 조회 |

## 시나리오 SC-1: A/S 담당자 출고 생성

```
A/S 담당자 → 티켓 상세(#4)에서 [출고 생성] 클릭
    ↓
출고 생성 패널 표시 (고객 정보 자동 채움)
    ↓
택배사: "CJ대한통운" 선택
송장번호: "6012345678901" 입력
CX 인계: 체크
    ↓
POST /api/shipments
  body: { ticketId, recipient, phone, address, carrier, trackingNo, cxHandover: true }
    ↓
토스트: "출고가 등록되었습니다"
    ↓
CS/CX에 인계 알림 발송
    ↓
출고 목록에 "발송" 상태로 표시
```

---

## 시나리오 SC-2: CS/CX 인계 확인

```
CS/CX 로그인 → 알림: "T-242 출고 건 CX 인계"
    ↓
A/S 출고 관리 → CX 인계 필터 활성화
    ↓
API: GET /api/shipments?cxHandover=pending
    ↓
해당 출고 건 확인 → [인계 확인] 클릭
    ↓
PATCH /api/shipments/{id}
  body: { cxHandoverStatus: "confirmed" }
    ↓
토스트: "인계 확인 완료"
    ↓
고객에게 배송 안내 이메일/SMS 발송 가능
    → POST /api/mail-logs/email → MailLog 기록
```

## API 엔드포인트 매핑

| 액션 | Method | Endpoint |
|------|--------|----------|
| 출고 목록 | GET | `/api/shipments?from&to&status&carrier&cxHandover` |
| 출고 생성 | POST | `/api/shipments` |
| 출고 수정 | PATCH | `/api/shipments/{id}` |
| 출고 삭제 | DELETE | `/api/shipments/{id}` (총괄만) |
| 택배 추적 | GET | `/api/shipments/{id}/tracking` |
| CX 인계 확인 | PATCH | `/api/shipments/{id}` |

# 확인 필요 사항

- **택배사 목록**: CJ대한/한진/로젠/우체국 외 추가 택배사 → 실무진 컨펌
