# 목적 & 핵심 가치

### 왜 만드나?

교환·반품의 접수→검수→처리 완료까지 전 과정을 추적하고, 회수된 제품의 **재고 재등록 / 리퍼브 / 부품 처리** 분기를 필수로 기록하여 재고 누락과 처리 사각지대를 제거한다.

| 구분 | 기존 시스템 | 신규 시스템 |
|------|------------|------------|
| 교환·반품 접수 | 어드민에서 수기 메모 기반 처리 | 체계적 접수 폼 + 상태 추적 |
| 검수 기록 | 별도 시트 또는 구두 전달 | 검수 결과 시스템 기록 (사진 첨부) |
| 처리 분기 | 담당자 재량, 기록 미비 | 재고 재등록/리퍼브/부품 분기 필수 선택 |
| 교환 출고 연결 | 수동으로 출고 건 별도 생성 | 교환 처리 시 신규 출고 자동 생성 |

# 접근 권한

| 역할 | 인원 | 접근 범위 | 제한 사항 |
|------|------|----------|----------|
| A/S 담당자 | 5명 | 접수·검수·처리 분기·출고 연결 전체 | 처리 완료 건 삭제 불가 |
| CS/CX | 4명 | 목록·상세 조회, 접수 생성 | 검수·처리 분기 불가 |
| 팀장 | 2명 | 전체 기능 + 처리 완료 건 수정 | 없음 |
| 총괄 | 1명 | 전체 기능 + 반품 정책 설정 | 없음 |

# UI 요소 상세 — 검수·처리 분기

| 요소 | 유형 | 설명 |
|------|------|------|
| 검수 결과 입력 | 폼 | 검수일, 검수자, 결과(정상/불량/부분불량), 상세 메모 |
| 검수 사진 첨부 | 파일 업로드 | 최대 5장, 검수 상태 증빙용 |
| 처리 분기 선택 | 라디오 버튼 (필수) | **재고 재등록** / **리퍼브** / **부품 처리** — 미선택 시 완료 불가 |
| 재고 재등록 | 서브 폼 | 원래 SKU로 재등록, 수량·상태 입력 |
| 리퍼브 | 서브 폼 | 리퍼브 SKU 지정, 리퍼브 등급(A/B/C) 선택 |
| 부품 처리 | 서브 폼 | 분해 부품 목록 입력, 각 부품별 재고 반영 여부 |
| 교환 출고 자동 생성 | 버튼 | 교환 유형에서 검수 완료 시 활성화, 클릭 시 Shipment 자동 생성 |

# 시나리오 1: 교환 접수 → 검수 → 출고

```
CS/CX               A/S 담당자              시스템
  │                    │                      │
  ├─ 교환 접수 생성 ───→│                      │
  │  (고객+제품+사유)    │                      │
  │                    │                      │
  │                    ├─ 회수 제품 도착        │
  │                    ├─ 검수 시작 ──────────→│
  │                    │  (상태→검수중)         │
  │                    │                      │
  │                    ├─ 검수 완료 입력 ──────→│
  │                    │  (결과: 불량 확인)      │
  │                    │  (사진 첨부)           │
  │                    │                      │
  │                    ├─ 처리 분기 선택: 리퍼브 →│
  │                    │  (리퍼브 등급: B)       │
  │                    │                      │
  │                    ├─ 교환 출고 생성 ──────→│
  │                    │                      ├─ Shipment 자동 생성
  │                    │                      ├─ 상태→처리완료
  │                    │                      ├─ MailLog 기록
  │                    │←──── 출고번호 반환 ────┤
```

# API 엔드포인트 매핑

| 메서드 | 엔드포인트 | 설명 | 권한 |
|--------|-----------|------|------|
| GET | `/api/exchanges` | 교환·반품 목록 조회 (필터·페이징) | 전체 |
| GET | `/api/exchanges/{id}` | 교환·반품 상세 조회 | 전체 |
| POST | `/api/exchanges` | 교환·반품 신규 접수 | CS/CX 이상 |
| PATCH | `/api/exchanges/{id}` | 상태 변경 (검수중/처리대기/완료/취소) | A/S 담당자 이상 |
| POST | `/api/exchanges/{id}/inspect` | 검수 결과 등록 (사진 첨부 포함) | A/S 담당자 이상 |
| POST | `/api/exchanges/{id}/restock` | 처리 분기 실행 (재고재등록/리퍼브/부품) | A/S 담당자 이상 |
| POST | `/api/exchanges/{id}/ship` | 교환 건 신규 출고 자동 생성 | A/S 담당자 이상 |
| GET | `/api/exchanges/stats` | 교환·반품 통계 (유형별/상태별 집계) | 팀장 이상 |

# 확인 필요 사항

1. 반품 시 환불 프로세스 범위 (자동 환불 vs 수동 승인?)
2. 리퍼브 등급 기준 정의 (A/B/C 각 조건)
3. 교환 시 동일 모델만 가능한지, 상위 모델 교환도 허용하는지
4. 부품 처리 시 폐기 부품의 비용 처리 (CostEntry 연동 여부)
5. 검수 사진 보관 기간 및 저장소 정책
